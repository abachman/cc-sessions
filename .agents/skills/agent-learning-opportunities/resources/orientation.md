# Repo Orientation: ccsessions

> Generated by /orient:orient. Re-run to update.

## One-line purpose
`ccsessions` is a terminal UI for browsing Claude Code session history for the current working directory so developers can search and inspect past agent conversations and tool activity.

## Primary language(s)
Go

## Pipeline / workflow stages
1. Map the current working directory to the corresponding Claude project-history folder under `~/.claude/projects`.
2. Discover session `.jsonl` files and parse their records into normalized `Session` and `Entry` data.
3. Classify raw log records into viewer-friendly kinds such as prompts, assistant text, tool calls, tool results, progress updates, and meta events.
4. Build the Bubble Tea model with searchable session state plus list and detail panes derived from the parsed sessions.
5. Render the TUI and handle filtering, focus changes, scrolling, and transcript formatting for the selected session.

## Core concepts
**Project history mapping**: The app does not read arbitrary files; `projectHistoryDir` in `internal/claude/sessions.go` converts the current repo path into the matching folder under `~/.claude/projects`.

**Session normalization**: `ParseSessionFile` and the `normalize*` helpers in `internal/claude/sessions.go` turn heterogeneous raw JSONL records into a consistent `Session` transcript.

**Entry kinds**: The `EntryKind` constants in `internal/claude/sessions.go` are the bridge between raw Claude log structure and the viewer's display logic.

**Searchable session summaries**: The parser derives `Summary` and `SearchText` fields so the UI can filter sessions by user-visible meaning instead of raw file names.

**Bubble Tea pane focus**: `internal/ui/model.go` models search, list, and detail panes as separate focus targets, which drives keyboard behavior and visual emphasis.

## Common gotchas
- `projectHistoryDir` in `internal/claude/sessions.go` sanitizes the current path by replacing path separators with `-`; if the viewer shows no sessions, the first thing to check is whether the expected Claude history folder name matches that transformation.
- CI in `.github/workflows/build.yml` runs `go test ./...`, but this repo currently has no `*_test.go` files; a new developer might assume there is coverage when there is only a zero-test pass today.
- The viewer intentionally surfaces non-conversational records from Claude logs, including progress and metadata events, so transcript output can look noisier than a chat-only log unless you understand the taxonomy in `docs/claude-session-log-taxonomy.md`.
